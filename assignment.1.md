3\. basic ML workflow for a food delivery app like Swiggy ?



* ML Workflow:

&#x20;  Data → Preprocessing → Modeling → Evaluation → Deployment



* Data: Collect customer orders, delivery times, locations, and ratings.
* Preprocessing: Clean the data by removing errors and filling missing values.
* Modeling: Train a machine learning model to predict delivery times.
* Evaluation: Test the model to measure how accurate its predictions are.
* Deployment: Use the model in the app to show estimated delivery times to customers.





4\. Traditional Programming vs Machine learning approach towards spam messages ?



In traditional programming, developers would create fixed rules to identify spam messages. For example, messages containing words such as "win money," "free prize," or too many links might be marked as spam. However, spammers can easily change their wording to bypass these rules. In machine learning, the system is trained using thousands of examples of spam and non-spam messages. Instead of following manually written rules, the model learns patterns from the data and automatically improves its ability to detect spam messages, even when the wording changes. Therefore, traditional programming relies on predefined rules, while machine learning relies on learning from examples and data.
